# WhareScore UX Audit — 2026-04-11

**Tested:** desktop (1440×900) and mobile (390×844), both personas (renter + buyer), signed-in session (sdfds0310@gmail.com).
**Test address:** 10 Customhouse Quay, Wellington Central (multi-unit building — a deliberately hard case for unit-vs-building CV handling).
**Method:** Walked the site as a first-time user. Home → search → report → scroll all sections → persona toggle → header nav → footer links → map layers → mobile flow → paywall modal.

## Status summary (resolved 2026-04-11)

The bulk of this audit has been worked through in a single pass. A ✅ next
to an item below means the underlying code was patched; a ⚠️ means the
change is defensive only (root cause sits in data, not UI) and a 🟡 means
the item was investigated and left as-is because the behaviour was
already correct or the concern was inaccurate.

Unfixed items worth revisiting in a future session:

- **H3** — first-use onboarding tour / demo address hint on the homepage.
- **H5 / M6** — map pin clustering at low zoom; the CBD still renders
  dozens of pins with no simplification.
- **M1 / M2** — dedicated legend entries for POI pin colours and a
  tap-for-detail popup on those pins.
- **N8** — mobile home bottom-sheet empty state when expanded (the four
  shortcut tiles leave most of the sheet blank while also hiding the map).
- **R23** — data sanity pass on "3 Cable car stops within 800 m" (likely
  a GTFS classification issue in Wellington).

## CRITICAL — count of "risk checks" / "data sources" is inconsistent across the site

This is the single most visible inconsistency — users see a different number almost every screen.

| Number | Where it appears |
|---|---|
| 12+ | Homepage footer line, report footer line, mobile footer |
| 27 | Mobile homepage bottom-sheet tile ("27 Layers") |
| 29+ | Homepage feature tile, "Get Full Report" sub-line |
| 32 of 32 | Report header ("32 of 32 risk checks available") and confidence footer |
| 36 | Report category list ("36 risk checks available for this property") |
| 40+ | `/about` page, meta description, OG description |
| 100+ | Homepage upper line, `/help` FAQ answer |
| 5 layers / 8 layers / 1 layer | Map legend counter vs map "Layers" badge |

**Fix:** pick one phrase and one number. This alone undermines credibility.

**Status:** ✅ Unified around **40+**. All static copy now reads "40+ risk
checks" or "40+ official NZ government data sources"; the mobile
shortcut tile was renamed from "27 Layers" to "All Layers"; and the two
dynamic report counts now both show `totalWithBonus` with the wording
"N risk checks completed" (previously one said "N of M available" and
the other said "N risk checks available for this property").

## Top-5 fixes to ship first

1. **Fix the unit/building CV leak** (C1-C2). Everything downstream — mortgage calculator, yield, upsell copy — depends on this. Nothing else you fix will matter if apartment searches return an $80M valuation.
2. **Unify the "risk checks" count** to a single number and a single label across homepage, report header, report body, about page, and help page. Pick 40, or 36, or whatever is real — but say it once.
3. **Pre-select the paid tier when the user clicks the paid CTA** (P1). Clicking "Get Full Report — $9.99" must land on the Full Report option, not the free Quick Report.
4. **Reconcile the rent-trend contradiction** (C3) and put direction-of-good cues on every 0-100 score (R1). Right now the report literally contradicts itself twice and the scores have no meaning.
5. **Fix `/suburb-guides` 404** (B1) — it's linked in every footer on the site.
