## Follow-up data audit ‚Äî 4-city report comparison (resolved 2026-04-11, later pass)

After the UX audit above, a second pass ran `check_audit2.py` against the generated report JSON for Wellington, Auckland, Christchurch and Queenstown (Cathedral Square / Queen Street / Customhouse Quay / Queenstown CBD) and surfaced six more field-level issues. All six were worked through in this session.

| # | Status | Issue | Fix |
|---|---|---|---|
| D1 | üü° | WLG `hazards.flood_extent_label` rendered as `'Flood Hazard ÔøΩ Inundation'` | False positive. The bytes in the DB are `0xE2 0x80 0x93` = UTF-8 en-dash (U+2013). Actual label is `"Flood Hazard ‚Äì Inundation"`. The `ÔøΩ` was Python `repr()` on the Windows cp1252 console, not a DB encoding bug. No code change. |
| D2 | ‚úÖ | CHC + ZQN `planning.zone_category` NULL | CHC loader was hardcoding `category=None`; QLDC loader wasn't extracting a category at all. Added `_derive_zone_category()` helper in `data_loader.py` that infers Residential/Business/Rural/Industrial/Open Space/etc. from the zone-name text, and wired both CHC and QLDC loaders through it. Auckland still uses its GROUPZONE coded-value domain (unchanged). |
| D3 | ‚úÖ | ZQN `planning.zone_code` was the same string as `zone_name` ("High Density Residential Zone") | QLDC's ArcGIS feed has no distinct short code ‚Äî `ZONE` is the full name. Loader now maps `zone_code` to `Zone_Code`/`ZONE_CODE` (may legitimately be NULL) instead of duplicating the full name. |
| D4 | üü° | CHC + ZQN `hazards.flood_extent_label` / `flood_extent_aep` NULL | Not a bug. Cathedral Square (CHC) and central Queenstown are not in any flood-hazard polygon; DATA-LAYERS.md confirms both CHC and QLDC flood loaders exist and are loaded. WLG's Lambton Harbour area is in a WCC inundation polygon because of reclaimed land ‚Äî CHC/ZQN central CBDs are simply not. No code change. |
| D5 | ‚úÖ | CHC `liveability.crime_area_unit` NULL despite `crime_percentile=74.1` / `victimisations=455` | New migration `0051_crime_area_unit_fallback.sql` wraps `best_au.area_unit` in `COALESCE(..., v_sa2_name)` so the field never returns NULL when the SA2 is known. Same migration also adds `CASE WHEN category IS NULL THEN 1 ELSE 0 END` to the `district_plan_zones` `ORDER BY` so rows with a populated category win ties (complementing D2). |
| D6 | ‚úÖ | CHC + ZQN `liveability.walking_reach_10min = 0` even though commit `6604cbb` claimed to fix walking_reach for regional cities | That commit only patched the Valhalla isochrone branch in `walking_isochrone.count_stops_in_isochrone()`. The radius fallback (used whenever Valhalla is unreachable) still only summed `metlink_stops + at_stops` and ignored `transit_stops`. Radius fallback now unions all three tables for total / bus / train / ferry counts, matching the isochrone branch. |
